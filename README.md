# 🧠 Memory Bot — Telegram-бот с памятью беседы + API

AI-агент в **n8n**, который:
1. **Запоминает** сообщения в Telegram-чате (отдельно для каждого чата);
2. по команде `/summary` делает **структурированную выжимку** беседы;
3. по команде `/ask <вопрос>` отвечает **на основе истории** (в т.ч. «какое твоё мнение?», «что говорили про X?»);
4. в группах понимает обращение через **@упоминание** или reply — без команд;
5. отдаёт тот же «мозг» через **HTTP API** (`POST /webhook/ask`) — для сайта/внешних систем.

Развёрнут локально в **Docker (Docker Desktop)**, наружу для Telegram открыт через туннель **cloudflared**.

## 🚀 Управление (3 команды)

| Действие | Команда |
|---|---|
| **Запустить всё** | `bash ~/Desktop/n8n-memory-bot/start-bot.sh` |
| **Остановить всё** | `bash ~/Desktop/n8n-memory-bot/stop-bot.sh` |
| **Проверить, что чисто** | см. блок «Контроль» ниже |

**`start-bot.sh`** поднимает: Docker Desktop → контейнер n8n → туннель cloudflared,
сам подставляет свежий адрес туннеля в `WEBHOOK_URL` и печатает адреса.
**После запуска** (адрес туннеля новый!) — открой n8n → workflow → **выключи и снова
включи Active**, чтобы Telegram узнал новый адрес.

**`stop-bot.sh`** гасит туннель и контейнер n8n. Память сохраняется (том `n8n_data`).
Чтобы освободить и движок — выйди из Docker Desktop (меню-бар → Quit).

- Редактор n8n: **http://localhost:5678**
- API/webhook: `<адрес-туннеля>/webhook/ask`

## 🏗 Как это устроено (стек)

```
Telegram ──► cloudflared (HTTPS-туннель) ──► localhost:5678 ──► n8n (Docker) ──► Claude API
                                                                   │
                                                          память по chatId (том n8n_data)
```

| Компонент | Роль | Автостарт после ребута Mac |
|---|---|---|
| **Docker Desktop** | движок контейнеров | ✅ да (стартует при входе) |
| **Контейнер n8n** (`n8nio/n8n`) | сам бот, порт 5678, том `n8n_data` | ✅ да (`--restart unless-stopped`) |
| **cloudflared** | публичный адрес для Telegram | ❌ нет → нужен `start-bot.sh` |

> После перезагрузки Mac: контейнер поднимется сам, а туннель — нет (и адрес сменится).
> Поэтому правило: **после ребута → `start-bot.sh` + переключить Active.**

## 🔧 Первичная настройка (если разворачиваешь с нуля)

1. **Telegram-бот:** @BotFather → `/newbot` → токен. Чтобы бот видел все сообщения
   в группе: **@BotFather → `/setprivacy` → бот → `Disable`** (после этого
   пере-добавь бота в группу).
2. **Запусти стек:** `bash start-bot.sh` → открой http://localhost:5678 → создай
   локальный аккаунт (любой email+пароль).
3. **Импорт workflow:** «⋯» → Import from File → `workflow.json`. (Ключ Anthropic,
   `ALLOWED_USERS`, `BOT_USERNAME` уже внутри JSON.)
4. **Токен Telegram** (единственное, что вводится вручную): открой ноду
   **Telegram Trigger** → Credential → Create New → вставь токен → Save.
   Затем **Send Telegram** → выбери ту же Credential.
5. **Имя бота:** в ноде **Store & Route** вверху проверь
   `const BOT_USERNAME = 'твой_бот'` (без @) — нужно для @упоминаний в группе.
6. **Save → Active.**

> ⚠️ Память копится только когда workflow **Active** (продакшн-запуски), а не при
> ручном «Execute Workflow».

## ✅ Как пользоваться

**В личке:**
- обычный текст — бот молча запоминает;
- `/summary` — выжимка;
- `/ask <вопрос>` / `/q <вопрос>` — ответ **по истории** беседы / мнение;
- `/chat <вопрос>` — **свободный вопрос ВНЕ контекста** (ответ как у обычного AI);
  при этом твой вопрос **сохраняется в память** чата (учитывается в `/summary`);
- `/start` или `/help` — справка, твой Telegram ID и ID чата.

> Отличие: `/ask` отвечает строго по переписке, `/chat` — свободно на любую тему.
> Чтобы `/chat` была в меню «/»: @BotFather → `/setcommands` → `chat - Свободный вопрос`.

**В группе:** то же + без команд — **@твой_бот <запрос>** или **reply** на сообщение
бота (AI сам поймёт: выжимка это или вопрос).

**Через API (curl):**
```bash
curl -X POST '<адрес-туннеля>/webhook/ask' \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: ТВОЙ_СЕКРЕТ' \
  -d '{"question":"что обсуждали про бюджет?", "chatId":"123456789"}'
# → {"answer":"...","question":"...","historyCount":12}
```

**Через сайт:** открой `test-api.html`, впиши URL вебхука, секретный ключ и ID чата
(ID узнаёшь командой `/start` в нужном чате).

## 🌐 Веб-поиск (бот ходит в интернет)

В нодах Claude включён встроенный **веб-поиск Anthropic** (`web_search`): Claude
**сам решает**, когда искать. Для вопросов про актуальные события/цены/новости —
ищет в интернете и отвечает со свежими данными; для обычных вопросов
(«расскажи анекдот», выжимка по истории) — отвечает сразу, без поиска.

- Удобнее всего через `/chat` (свободный вопрос): `/chat ситуация с топливом в Ялте сейчас`.

**Требования:**
1. **Включить в аккаунте:** https://console.anthropic.com → **Settings → Privacy** →
   разрешить **Web Search** (без этого API вернёт ошибку).
2. **Стоимость:** **$10 / 1000 поисков** + токены. Лимит на запрос — `max_uses: 3`
   (в JSON-body нод Claude). Один вопрос обычно = 1–3 поиска.

**Где настроено:** в нодах **Claude (Telegram)** и **Claude (API)**, поле *Body → JSON*:
```
tools: [ { type: 'web_search_20250305', name: 'web_search', max_uses: 3 } ]
```
Ответ с веб-поиском приходит несколькими блоками — поэтому ноды **Format reply** и
**API response** собирают текст из всех `text`-блоков (а не только из первого).

> Отключить поиск — убери `tools: [...]` из JSON-body нод Claude.
> Ответы обрезаются — подними `max_tokens` (сейчас 1500) там же.

## 🧩 Архитектура workflow (14 нод)

```
Telegram Trigger → Store & Route ─(нужен AI?)─true→ Claude (Telegram) → Format reply → Send Telegram
                                  └─false→ (есть ответ?)─true→ Send Telegram (help/отказ)
                                                        └─false→ (обычное сообщение: запомнили, без ответа)

Webhook /ask → API build prompt → (API: доступ?)─ok→ Claude (API) → API response → Respond
                                                 └─нет ключа/chatId→ API error response → Respond
```

- **Store & Route** — кладёт сообщение в `workflowStaticData('global').chats[chatId]`
  и определяет намерение (summary / ask / **chat** / smart / help / store / denied).
  `/chat` отвечает свободно (вне истории) и сохраняет вопрос в память.
- Память — простая (вся история чата уходит в контекст модели), до 500 сообщений
  на чат. Для демо и чатов средней длины этого достаточно.

## 🔐 Память изолирована по чатам

У каждого чата своя отдельная история — личка, группа A, группа B не пересекаются
(ключ хранилища = `chatId`). Утечки между чатами невозможны.

- Для **Telegram** `chatId` подставляется автоматически — ничего вводить не надо.
- Для **API** `chatId` обязателен в запросе (узнать: `/start` в нужном чате →
  «ID этого чата»).

## 🔒 Ограничение доступа

**1. Telegram — белый список (нода Store & Route):**
```js
const ALLOWED_USERS = [ 123456789, 987654321 ];  // Telegram ID людей (не чатов!)
```
- Свой ID узнаёшь: `/start` боту. Пустой `[]` = открыто всем.
- Чужие: на команды — отказ, обычные сообщения игнорируются.
- ⚠️ Сюда идут **ID пользователей** (положительные), НЕ ID групп (отрицательные).
- После правки: **Save** (и при необходимости toggle Active).

**2. API — секретный ключ (нода API build prompt):**
```js
const API_KEY = 'придумай_свой_секрет';
```
- Клиент шлёт его в заголовке `x-api-key` или поле `key`. Без него — отказ.
- Это **твой** пароль, не ключ Anthropic. Anthropic-ключ — отдельно, в нодах Claude.

## 🛠 Траблшутинг

| Проблема | Решение |
|---|---|
| **«Credential … does not exist for type telegramApi»** | Токен не переносится между инстансами. Открой Telegram Trigger и Send Telegram → назначь/создай credential заново (Create New → токен → Save). |
| **Поле токена стало `__n8n_BLANK_VALUE_…`** | Поле пустое / в режиме Expression. Переключи на **Fixed** и вставь токен заново. |
| **Бот не отвечает после `start-bot.sh` / ребута** | Адрес туннеля сменился → открой workflow → **выключи и включи Active** (Telegram перерегистрирует webhook). Токены при этом НЕ трогаются. |
| **Бот не видит сообщения в группе** | @BotFather → `/setprivacy` → `Disable`, затем **пере-добавь** бота в группу. Проверь `BOT_USERNAME` в Store & Route. |
| **API возвращает «Доступ запрещён»** | Неверный/отсутствует `x-api-key` или не указан `chatId`. |
| **«model not found» от Anthropic** | Поставь актуальное имя модели из docs.anthropic.com/models в нодах Claude (поле `jsonBody`). |
| **Группа «забыла» историю** | Обычная группа стала супергруппой → сменился `chatId`. Память для нового ID начинается заново. |

**Важно:** перезапуск контейнера / Mac / `start-bot.sh` и toggle Active —
**НЕ требуют** повторного ввода токенов. Токен вводится заново только при создании
**нового** инстанса n8n (новая БД/том).

## 🌐 Про туннель (cloudflared) коротко

n8n работает на `localhost:5678` и виден только внутри Mac. Telegram-у нужен
публичный HTTPS-адрес. **cloudflared** делает исходящее соединение к Cloudflare,
получает публичный адрес `*.trycloudflare.com` и прокидывает запросы обратно на
localhost — **без открытия портов на роутере**.

- Бесплатный **quick-туннель** даёт **случайный одноразовый** адрес → меняется при
  каждом перезапуске (поэтому после `start-bot.sh` нужен toggle Active).
- Хочешь **постоянный адрес** — именованный Cloudflare-туннель (нужен аккаунт +
  домен) или деплой на сервере с доменом (тогда туннель не нужен вовсе).

## ⚙️ Тюнинг

- **Модель.** По умолчанию `claude-sonnet-4-6` (ноды Claude, поле `jsonBody`).
  Можно задать env `CLAUDE_MODEL` — workflow её подхватит.
- **На OpenAI.** В нодах Claude: URL → `https://api.openai.com/v1/chat/completions`,
  заголовок → `Authorization: Bearer <ключ>`, тело → формат
  `{model, messages:[{role:'system',...},{role:'user',...}]}`; путь ответа →
  `choices[0].message.content`.
- **Динамический доступ.** Можно добавить команды `/allow <id>`, `/deny`, `/users`
  (хранить список в `workflowStaticData`) — добавление юзеров без правки кода.
- **RAG / вектор-БД (v2).** Когда история перестанет влезать в контекст — Postgres +
  pgvector / Supabase / Qdrant: эмбеддинги при записи, поиск релевантных кусков
  при вопросе. Для текущих объёмов НЕ требуется.
